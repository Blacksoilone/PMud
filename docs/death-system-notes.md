# 死亡系统设计笔记

死亡不是 combat 的附属结果，而是一套基础制度。它决定：

- 如何可控地剥夺玩家利益
- 如何把玩家从失败、危险或非法状态迁移出去
- 如何区分已确认权益、未确认收益和临时收益
- 如何给予惩罚但不制造不可接受的挫败
- 如何让区域风险、奖励和玩家准备程度形成关系

死亡系统会被很多其它系统参考：副本失败、主动脱困、非法位置恢复、被捕、昏迷、流放、挑战失败，都可以看作死亡系统的变体。

## 基础流程

死亡应当是一个流程，而不是单个扣血事件。

```text
DeathEvent:
  冻结 actor
  结束当前 conflict
  清理 combat / temporary state
  分类玩家利益和临时收益
  根据区域危险度计算惩罚
  处理掉落、经验、金钱、耐久、恢复状态
  通过 FallbackPlaceResolver 移动到安全地点
  应用 death recovery
  输出结构化死亡结果
```

原则：

> 死亡是游戏规则允许的剥夺点，但剥夺必须可预期、可解释、可被区域危险度控制。

## 利益分层

死亡处理前，应先区分利益类型。

### 核心长期权益

普通死亡通常不应随机剥夺这些内容：

```text
主线关键进度
绑定身份权益
仓库资产
邮件资产
已完成成就
关键任务物品
```

等级、绑定装备和经验是否可被影响，由区域危险度决定，不应由普通死亡随机处理。

### 可惩罚长期资源

这些可以成为死亡成本，但应有明确规则和上限：

```text
经验
金钱
装备耐久
非绑定物品
消耗品
当前携带资源
恢复时间
位置成本
```

### 临时收益

这些最适合在死亡时清理：

```text
当前战斗未拾取掉落
副本 pending reward
副本内临时物品
押送目标
采集中的进度
未完成仪式
临时状态
仇恨和追踪上下文
```

原则：

> 死亡优先清理未确认收益，再处理可惩罚资源，最后才考虑影响更重的长期成长或装备。

## 区域危险度

开放世界区域危险度使用 5 档，编号为 D0-D4。

```text
D0: 无惩罚安全区
D1: 低危救助区
D2: 常规威胁区
D3: 高危冒险区
D4: 极危刷宝区
```

5 档足够表达安全区、城镇周边、主要奖励区、高危冒险区和极危刷宝区的差异，同时不会细到内容设计者和玩家难以理解。

D0-D4 只用于普通开放世界、公共区域和可长期留下世界痕迹的区域。副本是隔离 run，内部房间的开放世界危险度没有意义；副本应使用独立的副本难度和失败策略。

## D0：无惩罚安全区

适用场景：

```text
安全城镇核心区域
复活点本身
特殊保护环境
```

规则：

```text
无任何死亡惩罚
死亡通常不应自然发生
如果因特殊系统导致死亡，只做安全迁移或状态修复
```

设计目的：

```text
给玩家整理、社交、交易、恢复和重新规划的空间
作为 FallbackPlaceResolver 的常见目标
```

## D1：低危救助区

适用场景：

```text
大部分城镇周边
城墙上
开发过的狩猎场地
有人类/己方活动痕迹明显的森林或道路
```

设定解释：

```text
玩家死亡后可以被己方人员、巡逻队、猎户、守卫或路人救回。
救助不是免费，所以通过经验和救助费用体现损失。
```

规则：

```text
惩罚经验
经验不会因此降级
最多降到当前等级 0 经验
以支付救助费用的形式损失金钱
通常不损失装备或重要物品
移动到本区域安全点
```

设计目的：

```text
让死亡不是免费回城
保护新手和低危探索
让玩家理解死亡规则，但不制造长期挫败
```

## D2：常规威胁区

适用场景：

```text
出新手期后的主要冒险区域
玩家必须经常经过的危险地带
主要获得奖励的野外或地下区域
```

规则：

```text
大量惩罚经验
损失可观金钱
迫使玩家养成存钱习惯
损失部分非绑定物品
清理当前未确认收益
移动到本区域安全点或较近安全点
```

设计目的：

```text
这是主要有威胁的死亡档位
死亡应当痛，但不应摧毁角色
玩家需要评估是否继续深入、是否携带太多资源、是否该回城存钱
```

## D3：高危冒险区

适用场景：

```text
高危区域
深层危险地带
除特殊地方或介绍性任务外，玩家不会被迫经过的区域
进入这些区域被视为主动冒险
```

规则：

```text
损失经验可能导致等级下降
非绑定物品大量损失
绑定物品扣除大量耐久，甚至损坏
清理本次冒险收益
移动到较远或明确的安全点
死亡后不准备多套装备，不应能立刻在本区域继续正常活动
```

设计目的：

```text
让高危区域有真正压迫感
要求玩家准备备用装备、补给和撤退计划
让“冒险失败”成为需要恢复和调整的事件
```

D3 是经验与等级风险最强的档位之一。它惩罚的是玩家对角色成长和装备耐久的冒险投入。

## D4：极危刷宝区

适用场景：

```text
极危区域
必须有特殊许可、明确入口或强提示才能进入
玩家不应误入
进入时就应该做好丢失物品的准备
类似高风险刷宝区域
```

规则：

```text
所有物品都有极高概率损失
非绑定物品和携带物是主要风险对象
绑定物品也可能受到严重耐久损失或其它高成本损坏
经验惩罚比例低于 D3
清理本次冒险收益
移动到明确安全出口
```

设计目的：

```text
鼓励玩家平衡装备成本和性能
允许玩家用低成本装备降低死亡损失
让高收益刷宝建立在可主动管理的风险上
```

D4 的经验惩罚低于 D3，是因为玩家不能主动存储经验来规避损失。如果极危区域同时高概率损失装备和高比例损失经验，玩家几乎无法通过准备降低损失。

因此 D4 应将主要风险放在“玩家可主动管理的东西”上：装备成本、携带物价值、消耗品、是否携带备用装备。玩家可以选择低强度、低成本装备，以更高死亡概率换取更低损失；也可以选择高性能装备，提高效率但承担更高物品风险。

原则：

> D3 主要惩罚角色成长和高危冒险失败；D4 主要惩罚装备与携带物选择，用于高风险刷宝设计。

## 死亡与 FallbackPlaceResolver

死亡后的移动不应硬编码到某个全局复活点，而应使用 FallbackPlaceResolver。

推荐优先级：

```text
当前玩法 safe_respawn_place
当前区域 fallback_place / safe_respawn_place
玩家绑定复活点
全局默认安全点
```

不同危险度可以影响安全点距离：

```text
D1:
  回到本地区近处安全点

D2:
  回到本地区安全点

D3:
  可能回到更远的安全点

D4:
  回到明确安全出口，通常离极危区域入口有距离
```

## 死亡与副本

副本死亡按副本事务规则处理，不直接套用开放世界 D0-D4。

原因：副本是隔离 run。副本里的房间、怪物、机关、掉落和临时物品都属于当前 `DungeonRun`，离开或失败时整体结算/清理。一个副本内部房间即使看起来更危险，也不应该改变开放世界意义上的死亡掉落、地面清洁或复活距离规则。

副本应声明 run-level 的难度和失败策略：

```text
DungeonDifficulty:
  描述整个副本 run 的挑战强度、推荐人数、推荐等级、奖励倍率、怪物/机关强度

DungeonFailurePolicy:
  描述死亡、团灭、主动放弃、超时、断线超时后如何处理门票、pending reward、检查点和外部惩罚
```

副本房间内部可以有 encounter tuning，但不使用 D0-D4：

```text
DungeonRoomTuning:
  encounter_level
  trap_intensity
  boss_phase
  checkpoint_role
  resource_pressure
```

这些字段只影响副本内容编排，不决定开放世界死亡惩罚。

第一版默认：

```text
副本内死亡:
  rollback run 或按副本规则失败
  release reserved ticket，除非副本明确规定死亡也消耗资格
  不提交 pending rewards
  清理副本内临时物品
  移动到副本 safe_exit_place
  应用 death recovery
```

副本可以后续定义更硬的规则，例如死亡扣除挑战次数、消耗门票、保留部分进度或从副本内检查点复活。但这些是副本内容规则，不是默认死亡规则。

断线超时也可以成为副本失败条件。副本本身是隔离 run，所以断线后长时间未连接时，不需要伪装成普通开放世界死亡；它可以直接由 `DungeonFailurePolicy` 声明为 run failed、checkpoint failed、ticket consumed 或 death-equivalent。

示例：

```text
story_dungeon:
  difficulty: story
  failure_policy:
    death: checkpoint_respawn
    ticket: release_on_failure
    pending_reward: clear_unsettled

challenge_dungeon:
  difficulty: challenge
  failure_policy:
    death: fail_run
    ticket: consume_on_entry_or_failure
    pending_reward: clear_unsettled

hardcore_dungeon:
  difficulty: hardcore
  failure_policy:
    death: fail_run
    ticket: consume_on_entry
    external_penalty: explicit_content_rule
```

原则：

> 开放世界死亡由 WorldDangerLevel D0-D4 控制；副本死亡由 DungeonDifficulty 和 DungeonFailurePolicy 控制。副本房间没有开放世界危险度，只有副本内部调参语义。

## 死亡与主动脱困

主动脱困可以被视为玩家主动触发的轻死亡迁移，但惩罚更轻。

```text
escape / unstuck:
  允许脱离危险或 bug 状态
  立即移动到安全点
  清理临时收益和可滥用状态
  给予传送后限制和 cooldown
```

死亡与主动脱困的区别：

```text
死亡:
  根据危险度剥夺经验、金钱、物品或耐久

主动脱困:
  主要剥夺当前位置收益、行动节奏和一段时间自由
```

主动脱困即使能脱离危险也是可以接受的利用，只要时间限制、cooldown 和收益清理让它不能成为免费回城或稳定逃生循环。

## 死后世界

死亡后不一定要立即复活。可以把死后世界做成普通地图，而不是特殊运行时。死后世界也不必只有一个；不同文明、地区、信仰、种族或特殊玩法可以通向不同的死后地图。

```text
死亡:
  actor 进入 dead / spirit 状态
  移动到死后世界中的某个 Place
  玩家在死后世界内通过普通命令、普通房间、普通 NPC、普通门进行互动
  满足条件后通过“回到正常世界的门”转生
```

这样死后世界仍然使用现有空间、命令、物品、门和条件系统，不需要专门做一套 afterlife VM。

死亡时应通过 `AfterlifeResolver` 选择死后目的地：

```text
AfterlifeResolver context:
  death_place
  area / region
  civilization / culture
  deity / faith
  race / faction
  danger_level
  active instance / dungeon / special rule
  player bindings or curses

AfterlifeResolver result:
  afterlife_place
  psychopomp / death-god NPC
  reincarnation gates
  afterlife_cost_model
  available loss-reduction paths
```

例子：

```text
中原文明地区死亡:
  进入阴司地图
  由城隍、判官、鬼差处理转生

北方蛮族地区死亡:
  进入英灵厅或寒原死境
  由战死者、祖灵或异族死神处理转生

海洋文明地区死亡:
  进入沉没港湾或冥海
  由摆渡人、海神使者或溺亡者引路

特殊邪教区域死亡:
  进入该邪神的死后领域
  转生条件可能被扭曲或更昂贵
```

这样可以让不同文明拥有自己的死亡叙事和死神，同时仍然复用同一套地图、门、NPC、物品和条件系统。

危险度决定死亡的基础严重程度，但死后世界决定死亡代价的具体形态。同样是 D2 或 D3，在不同文明中也可以有完全不同的死亡成本。

例子：

```text
悍不畏死的文明:
  死后世界友善
  死神或祖灵乐于放人回来
  转生流程短，金钱和物品损失较低
  这个文明的人不怕死，是因为死亡的制度成本确实较低

贪婪死神控制的文明:
  死后世界像债务市场或审判场
  转生需要缴纳高额赎金、献祭物品或接受契约
  死亡代价沉重且社会广泛畏惧死亡

尚武文明:
  战死者可能得到较宽容的转生条件
  逃亡者或怯战者死亡代价更高

商贸文明:
  死亡代价可能更多体现为金钱、债务、保险和赎身凭证

诅咒地区:
  危险度不一定最高，但死后世界恶劣，转生条件被扭曲
```

因此死亡代价应由多部分合成：

```text
DeathCost = DangerLevel base penalty
          + Afterlife cost model
          + Activity / instance modifiers
          + Player protection / curse modifiers
```

框架层不应把 D0-D4 直接写死成唯一代价表。D0-D4 提供基础风险等级；AfterlifeResolver 和死后世界内容提供文化、信仰、死神和地区差异。

死后损失减免物品也可以自然表达为普通内容：

```text
引魂符:
  在死后世界中作为通向正常世界的钥匙
  使用后减少金钱或经验损失

护命符:
  死亡后让玩家出现在更近的死后入口
  或开启一扇代价较低的转生门

赎身凭证:
  可支付给死后世界 NPC
  换取减少物品损失或缩短死后流程
```

本质上，死亡惩罚可以被设计成“从死后世界返回正常地图所需的条件”。这些条件可以是：

```text
支付金钱
消耗道具
选择放弃部分物品
接受经验损失
接受装备耐久损坏
完成一段低风险死后流程
等待一段时间
```

原则：

> 死后世界是普通地图；可以有多个死后世界，由 AfterlifeResolver 按死亡上下文选择。危险度决定基础严重程度，死后世界决定死亡代价的具体形态。转生是从死后地图回到正常地图的受条件限制移动。影响死亡损失的物品，本质上是改变死后世界通行条件的钥匙、凭证或替代支付手段。

## 当前结论

```text
死亡必须造成损失，否则会变成便捷移动、回血、回资源手段。
开放世界死亡惩罚由区域危险度控制。
D0 无惩罚。
D1 经验不降级 + 救助费用。
D2 大量经验 + 可观金钱 + 部分非绑定物品。
D3 可能降级 + 大量非绑定损失 + 绑定装备严重耐久/损坏。
D4 物品极高风险 + 经验惩罚低于 D3，用于高风险刷宝。
死亡移动使用 FallbackPlaceResolver。
副本死亡按副本事务处理，使用独立的 DungeonDifficulty / DungeonFailurePolicy，不套用 D0-D4。
断线默认先给短暂重连窗口；长时间未连接是否死亡，由离线可利用性决定。如果玩家离线会显著获利或规避代价，应由战斗、副本、特殊环境或特殊状态显式声明为失败或死亡。高危区域本身不构成断线死亡条件。

高危区域断线只是延后风险：玩家下次上线仍在该区域，仍然要面对原本的环境危险。即使断线期间找强大玩家来清理附近怪物，也不是断线本身创造的显著收益，因为这类协作原本就可以通过组队或支援获得。

战斗、副本和特殊状态不同：这些场景里离线可能跳过正在发生的伤害、限时、团灭、资源消耗、机关结算或死亡结算，因此可以显式规定断线超时等同失败或死亡。
主动脱困参考死亡迁移，但惩罚更偏行动限制和收益清理。
死后世界可以是复数普通地图，AfterlifeResolver 决定进入哪个死后世界；转生门和死后通行条件负责表达死亡结算。
同一危险度在不同文明、信仰或死神控制下可以有不同死亡代价。
```

后续还需要讨论：

```text
绑定物品如何定义
非绑定物品损失是掉落、销毁、还是可找回尸体
经验损失的具体公式
金钱损失是否只扣随身现金
装备耐久损坏是否可修复
death recovery 的具体限制
不同危险度如何在开放世界房间、区域中声明和展示
DungeonDifficulty / DungeonFailurePolicy 如何声明和展示
AfterlifeResolver 如何根据地区、文明、信仰、种族和特殊状态选择死后世界
死后世界入口、转生门、减免物品、死神 NPC 和死亡结算条件如何配置
```
