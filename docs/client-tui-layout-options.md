# TUI 客户端布局草案

本布局已定档为“大终端固定客户端”方向。

对齐约定：

```text
ASCII 字符显示宽度为 1。
中文字符显示宽度为 2。
图示按严格中英 2:1 等宽字体绘制。
实际客户端应使用 wcwidth 或等价宽度计算进行布局。
```

本文中的字符画只表达布局意图，不作为最终对齐真源。真实客户端需要一个字符画布局渲染器，按面板树、显示宽度、图层和内容自动生成边框、分隔线、地图连接线和 overlay。

核心要求：

```text
1. 地图常驻。
2. 房间显示常驻。
3. 个人状态常驻。
4. 环境信息常驻。
5. LOG 是主要弹性滚动区域。
6. 选择框只在需要时弹出，竖排显示，可滚动。
7. 背包、装备等页面使用居中 overlay，不常驻。
8. 所有操作原则上都能通过命令完成，快捷键只是效率层。
```

## 字符画布局渲染器

手工维护字符画很容易出错。真实客户端应有一个统一的布局渲染器，而不是每个界面手写边框和空格。

渲染器输入：

```text
LayoutTree:
  root width / height
  panels
  split rules
  fixed / flexible sizes
  z-index layers

Panel:
  id
  x, y, width, height
  border style
  title
  content renderer

Layer:
  base layout
  popup selection
  overlay inventory/equipment/settings
```

渲染器职责：

```text
按 wcwidth 计算显示宽度
根据面板矩形生成边框和分隔线
处理交叉点、T 字连接和角点
按宽度截断或 padding 内容
支持中文全角 / ASCII 半角混排
支持 overlay 覆盖底层布局
支持 popup 只覆盖指定区域
```

这意味着：

```text
地图面板不手写空格。
选择框不手写嵌套边框。
Overlay 不手写背景。
所有图层都由同一个 compositor 合成。
```

### 3x3 地图渲染

地图应由地图数据渲染，而不是手写字符画。

```text
Map3x3:
  cells[3][3]
  current cell
  links between neighboring cells
  optional diagonal links
```

每个 cell 有固定显示宽度，名称在 cell 内居中。

```text
cell_width: 8 or 10 display columns
horizontal_gap: connection columns
vertical_gap: connection rows
```

地图渲染器负责生成：

```text
cell name rows:
  房间名居中

horizontal connector rows:
  用 --- 或 -- 连接左右房间

vertical connector rows:
  用 | 连接上下房间

diagonal connector rows:
  用 / 或 \ 表示斜向连接
```

示意：

```text
  松林       山路       城门
    |          |          |
  官道 ---  *渡口 ---  码头
    |        / |          |
  林径       渡船       河心
```

这张图只是渲染目标示意。实际对齐应由 Map3x3 renderer 按 cell_width 和 wcwidth 生成。

### Overlay 合成

Overlay 不应该重新手写一份背景。正确流程是：

```text
render base layout
render overlay panel into a rectangle
compose overlay over base buffer
```

因此底层的横向分界线、纵向分界线、状态栏、环境栏都应该自然存在；overlay 只覆盖它占据的矩形区域。

如果 overlay 半透明不可用，使用普通覆盖即可：被覆盖区域暂时不可见，关闭 overlay 后 base layout 原样恢复。

### Popup 合成

选择框也是一个 layer，但它不是全屏 overlay。

```text
popup target area:
  LOG right side

popup behavior:
  覆盖右侧 LOG viewport
  不覆盖 ENV
  不覆盖输入栏
  不改变 LogBuffer
```

Popup 关闭后，LOG 右侧按当前 viewport 重新渲染。

### 全屏字符画模式

除了常规 HUD 布局，还需要一种特殊 LayoutMode：整个屏幕除输入框外都用于显示一张字符画或观景画面。

适用场景：

```text
登录 / 标题画面
角色创建
章节过场
观赏美景的地点
梦境、死后世界、神殿等仪式性场景
大型地图或壁画展示
```

模型：

```text
LayoutMode: full_screen_art

ArtPanel:
  占据除输入区外的全部可用空间
  支持居中、裁剪、缩放档位或滚动
  可显示静态 ASCII / ANSI art
  可叠加少量标题、说明或候选提示

InputPanel:
  仍固定在底部
  保留命令输入能力
```

示意：

```text
+--------------------------------------------------------------------------------------------------------------------+
|                                                                                                                    |
|                         __  __           _      __        __             _     _                                   |
|                        |  \/  |         | |     \ \      / /            | |   | |                                  |
|                        | \  / |_   _  __| |      \ \ /\ / /__  _ __ ___ | | __| |                                  |
|                        | |\/| | | | |/ _` |       \ V  V / _ \| '__/ _ \| |/ _` |                                  |
|                        | |  | | |_| | (_| |        \_/\_/ (_) | | | (_) | | (_| |                                  |
|                        |_|  |_|\__,_|\__,_|             \___/|_|  \___/|_|\__,_|                                  |
|                                                                                                                    |
|                                      晚钟响起，远处的城灯一盏盏亮起。                                              |
|                                                                                                                    |
+--------------------------------------------------------------------------------------------------------------------+
|>                                                                                                                   |
+--------------------------------------------------------------------------------------------------------------------+
```

原则：

```text
full_screen_art 是一种 layout mode，不是普通 overlay。
它可以隐藏地图、房间、状态、环境和 LOG。
输入框仍保留，确保玩家可以继续输入命令或退出。
退出该模式后恢复常规 HUD 布局。
```

如果观景点需要长时间停留，可以由服务器发送 `PresentationEvent(channel=room/art, layout_mode=full_screen_art)`，客户端进入该模式；玩家移动、输入 `look normal`、按 Esc 或场景结束后恢复常规布局。

区域含义：

```text
地图:
  常驻区域字符画，类似北侠的区域图。第一版按 3x3 矩阵显示当前局部区域。

房间:
  常驻当前房间观察结果。

状态:
  常驻玩家关键资源和状态。

环境:
  常驻压缩环境信息，可用颜色表达天气、时间、危险、区域状态。

日志:
  主要动态区域，显示行动结果、聊天、战斗、系统提示等历史内容。

输入:
  固定在底部。
```

## 选择框

选择框只在存在候选项、确认请求或临时菜单时显示。选项竖排，过长时内部滚动。

选择框覆盖 LOG 的右侧，保留左侧日志和左侧环境栏。关闭后恢复原显示。

```text
+----------------------------+----------------------------------------------------------+----------------------------+
|地图                        |房间                                                      |状态                        |
|  松林      山路      城门  |东渡口                                                    |气血      82 / 100          |
|  官道     *渡口      码头  |一座木制渡口伸入昏暗的河水。                              |内力      31 /  45          |
|  林径      渡船      河心  |                                                          |体力      64 /  80          |
|                            |                                                          |危险      D2                |
|                            |                                                          |                            |
+----------------------------+----------------------------------------------------------+----------------------------+
|环境                        |日志                                        |选择目的地                                |
|时辰  黄昏                  |船夫说：“去洛阳，五十文。”                  |> 1. 洛阳驿站          50 文              |
|天气  微风                  |你没有船票。                                |  2. 开封驿站          80 文              |
|光照  昏暗                  |                                            |  3. 扬州码头         120 文              |
|区域  已开发野地            |                                            |                                          |
|                            |                                            |Enter 确认  Esc 取消  ↑↓ 移动             |
+----------------------------+--------------------------------------------+------------------------------------------+
|> ride ferry                                                                                                        |
+--------------------------------------------------------------------------------------------------------------------+
```

选择框规则：

```text
↑ / ↓:
  移动选择

PageUp / PageDown:
  选择框内部滚动

Enter:
  确认当前项

Esc:
  取消

Tab:
  在输入、选择框、LOG 等焦点区域切换
```

快捷键最终发送结构化 command envelope，例如：

```text
kind: candidate_select
request_id: travel_123
candidate_id: luoyang_station
```

## Overlay

背包和装备不常驻。查看时用居中大框覆盖其它显示，类似设置界面；关闭后回到主界面。

Overlay 背景仍绘制基准布局，表示它是覆盖在当前界面之上，而不是切换到另一套屏幕。

```text
+--------------------------------------------------------------------------------------------------------------------+
|地图                        |房间                                                      |状态                        |
|  松林      山路      城门  |东渡口                                                    |气血      82 / 100          |
|  官道     *渡口      码头 +------------------------------------------------------------+内力 31/45                 |
|                           |背包                                                        |体力 64/80                 |
|                           |                                                            |饥饿 无                    |
|                           |  1. 铁剑                 未装备                            |口渴 轻微                  |
|                           |  2. 船票                 可使用                            |状态 正常                  |
|                           |  3. 干粮 x3              食物                              |危险 D2                    |
|                           |  4. 引魂钱               死后通行物                        |                           |
|                           |                                                            |                           |
|                           |Enter 查看/使用   Tab 分类   Esc 关闭                       |                           |
|                           +------------------------------------------------------------+                           |
|环境                        |日志                                                                                   |
|时辰  黄昏                  |船夫说：“去洛阳，五十文。”                                                             |
+--------------------------------------------------------------------------------------------------------------------+
```

装备页面可以使用同样 overlay，但具体布局暂不决定。

## 响应式规则

固定客户端可以要求最小终端尺寸。低于最小尺寸时，不应强行挤压到不可读，而应提示玩家增大终端。

```text
低于最小尺寸:
  显示 resize required 画面
  告知当前尺寸和建议最小尺寸
  保留退出、重连、设置命令

达到最小尺寸:
  显示地图、房间、状态、环境、日志、输入
  选择框和 overlay 正常可用

超过最小尺寸:
  优先把额外空间给 LOG
  其次增加房间可见行数
  地图、状态、环境使用固定宽度或少数档位
```

面板尺寸策略：

```text
地图:
  固定宽度或 2-3 个档位
  第一版按 3x3 矩阵显示，房间名在固定格内居中

状态:
  固定宽度
  只显示关键资源和状态

环境:
  固定宽度，与地图对齐
  使用压缩字段和颜色表达环境

房间:
  高度可有少数档位
  过长描述可折叠、滚动或通过 look detail 展开

日志:
  主弹性区域
  终端变大时主要由 LOG 填满

输入:
  固定高度，必要时支持多行输入
```

原则：

> 其它面板保持稳定，LOG 吃掉大部分可变空间。这样 resize 时玩家不会失去空间感，而动态历史内容获得更多滚动容量。

## LOG 滚动

LOG 是主要动态变化区域，内容长度近似无限，因此必须支持独立滚动浏览。

推荐模型：

```text
LogBuffer:
  保存最近 N 个 PresentationEvent
  按当前 LOG 宽度重新渲染和 wrap

Viewport:
  follow_tail: true/false
  scroll_offset_from_bottom
  visible_line_range
```

默认状态：

```text
follow_tail = true:
  新日志到来时自动滚到底部
```

玩家向上滚动时：

```text
PageUp / Ctrl+Up:
  follow_tail = false
  LOG 停留在历史位置
  新日志继续进入 buffer，但不强行跳到底部
```

有新日志到来且玩家不在底部时：

```text
显示提示:
  -- new messages below --
  或右下角显示 +3
```

回到底部：

```text
End / Ctrl+End:
  follow_tail = true
  显示最新日志
```

建议按键：

```text
PageUp / PageDown:
  翻页滚动 LOG

Ctrl+Up / Ctrl+Down:
  小步滚动 LOG

End:
  跳到底部并恢复 follow_tail

Home:
  跳到当前 buffer 顶部

Alt+L 或 Ctrl+L:
  聚焦 LOG / 取消聚焦
```

输入体验原则：

```text
玩家输入命令时不应被 LOG 滚动打断。
如果 LOG 不在底部，新消息不应抢走光标。
候选框或 overlay 打开时，LOG 仍然继续收消息，但不一定自动滚动。
```

日志数据应保留结构化来源：

```text
每条日志应能追溯到 PresentationEvent id / channel / priority。
未来可以支持按 channel 过滤，例如 combat/chat/system。
```

## 当前基准

```text
地图独立常驻，按 3x3 固定格居中显示。
房间居中常驻。
状态和环境常驻。
LOG 是主弹性滚动区域。
选择框按需弹出，覆盖 LOG 右侧，竖排滚动。
背包和装备用 overlay。
```

后续需要继续细化：

```text
最小终端尺寸
地图面板宽度档位
房间面板高度档位
LOG channel 过滤
选择框覆盖位置和焦点规则
Overlay 与 LOG / Selection 同时存在时的优先级
```
